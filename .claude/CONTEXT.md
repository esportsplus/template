## Project Overview

`@esportsplus/template` is a compiler-optimized HTML/SVG templating library for JS/TS: template
literals are transformed at **build time** (via a Vite plugin or a tsc/ts-patch transformer) into
optimized DOM-manipulation code, so there is zero runtime template parsing. It integrates with
`@esportsplus/reactivity` for reactive slot updates, uses `@esportsplus/queue` for object pooling,
and `@esportsplus/typescript` for compiler-side AST utilities. Design choices called out in the
README: array reordering (sort/reverse) uses the `moveBefore` DOM API when available to preserve
element state (focus, animations, iframe content), falling back to `insertBefore`; events are
delegated by default for efficiency, with a fixed set of non-bubbling events attached directly;
async effect slots support an optional fallback callback for loading states; dev-mode HMR
fine-invalidates only the affected template factories on save rather than reloading the page.
Published as dual entrypoints (`.` and `./compiler`, `./compiler/tsc`, `./compiler/vite`) so
consumers import the runtime and the build-tool transformer separately.

## Build & Dev

`build` runs `tsc` — a straight TypeScript compile to `build/` (no bundler), matching the
`types`/`main`/`exports` fields in package.json. `prepare` and `prepublishOnly` both re-run
`pnpm build`, so a fresh install or `pnpm publish` always ships compiled output. `test` runs the
suite once via `vitest run`; `test:watch` runs `vitest` in watch mode for local iteration;
`test:coverage` adds `--coverage` (backed by `@vitest/coverage-v8`). `bench:run` runs
`vitest bench` against the separate `vitest.bench.config.ts`, keeping benchmark config (and the
`bench/` suite) isolated from the regular `vitest.config.ts` test run.
